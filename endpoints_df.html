II. APIs de Processos e Candidatos

1. Consulta de Processo (Endpoint Valid/Superprati)
Endpoint
POST /importacaodf/consultarprocesso

Esta API consulta o processo de uma pessoa no ambiente Detran-DF.

1.1. Parâmetros da Requisição (Body JSON)
NomeDescriçãoValidação
CpfCPF da pessoa cujo processo deve ser consultado.Obrigatório, não pode ser vazio.
CnpjCNPJ do centro de formação vinculado ao processo.Obrigatório, não pode ser vazio.
TipoProcessoTipo do processo a ser consultado.Obrigatório. Valores: "T" (Teórico) e "P" (Prático).
1.2. Padrão de Resposta (Status HTTP e Corpo JSON)

O Corpo de Resposta JSON inclui Código, Mensagem (resultado da consulta) e Entity (dados solicitados).

CódigoMensagem (Possível Retorno)
0OK
5Detectada irregularidade cadastral na chave de acesso
6Campo 'Tipo de processo' está incorreto.
8Processo não encontrado no Detran
99Problema ao consumir API do Detran
2. Consultar Candidato (Endpoint Credenciadas/VSoft)
Endpoint
POST /api/cfc/buscar

Consulta as informações dos candidatos que poderão fazer aulas teóricas/práticas nos CFCs, integrando-se com a VSoft e a base do Detran-DF.

2.1. Parâmetros de Envio (Body JSON)
NomeDescriçãoValidação
cpfCPF do Candidato.É obrigatório.
tipoPesquisaDefine o Tipo de Curso a ser consultado.Valores aceitos: T (Teórico), P (Prático), S (Simulador).
2.2. Campos de Retorno Comuns na Data

Os dados de retorno são influenciados pelo tipoPesquisa.

  • renach, nomeCandidato, dataNascimento, motivoRequerimento.
  • pne: Indica se o candidato é Portador de Necessidades Especiais (PNE).
  • dataValidadeExameMedico.
  • Retorno Específico (Tipo Pesquisa T - Teórico): Retorna a lista dadosAulasTeoricas.
  • Retorno Específico (Tipo Pesquisa P - Prático): Inclui dataEmissaoLADV e dataValidadeLicencaLADV.
3. Notificação de Captura Biométrica
Endpoint
POST /biometria

Dispara a sincronização com a Valid das informações biométricas da pessoa com o CPF informado.

3.1. Parâmetros da Requisição (Body JSON)
NomeObrigatoriedade
CpfObrigatório, não pode ser vazio.
RenachOpcional.
3.2. Possíveis Retornos HTTP
CódigoDescrição
200 OKO processo de sincronização foi iniciado com sucesso (corpo vazio).
400 Bad RequestAlguma regra de negócio violada.
401 UnauthorizedToken de autenticação não informado ou inválido.

III. APIs de Envio de Aulas (Credenciadas/VSoft)

1. Enviar Aula Teórica
Endpoint
POST /api/aula/envio-aula-teorica

Envio de aula teórica para validação e inclusão.

1.1. Parâmetros de Envio (Body JSON)

Todos os campos são obrigatórios.

NomeDescriçãoFormato / Valores Possíveis
cpfAlunoCPF do aluno.Somente números.
resultadoResultado da aula."1" (Validar), "0" (Cancelar).
codigoDisciplinaCódigo da disciplina.01 a 05 (Direção defensiva, Primeiros socorros, etc.).
dataInicioData de início da aula.yyyyMMddHHmmss (Ex: 20250131080000).
dataFinalData final da aula.yyyyMMddHHmmss (Ex: 20250131085000).
1.2. Observações sobre o Retorno

O retorno fornece um codigoRetorno sobre a operação.

Código RetornoMensagem
00Dados gravados com sucesso.
35Dados gravados com sucesso e carga horária concluída.
01Número de RENACH Inválido.
99Aplicação fora do ar/falha de conexão.
2. Enviar Aula Prática
Endpoint
POST /api/aula/envio-aula-pratica

Envia a aula prática para inclusão direta na base do Detran-DF.

2.1. Parâmetros de Envio (Body JSON)

Todos os campos são obrigatórios.

NomeDescriçãoFormato / Valores Possíveis
cpfAlunoCPF do aluno.Somente números.
placaAulaPlaca do veículo.-
dataAulaData da aula.Formato yyyyMMdd.
horaInicioHorário do início da aula.Formato hhmm (Ex: 0800).
2.2. Observações sobre o Retorno

O retorno fornece um codigoRetorno (e mensagem vazia no exemplo de sucesso).

Código RetornoDescrição
996SERVIDOR FORA DO AR.
997ERRO NO PROCESSAMENTO.
998AULA JA CADASTRADA, CANDIDATO NAO CADASTRADO, etc.

IV. APIs de Consulta de Recursos (Valid/Sincronização Externa)

1. Consulta Dados do CFC
Endpoint
GET sincronizacaoExterna/centroFormacao

Este método lista todos os CFCs.

2. Consulta Dados de Instrutor
Endpoint
GET sincronizacaoExterna/centroFormacao/{codigoCfc}/instrutores

Este método lista todos os instrutores de um CFC.

3. Consulta Salas Teóricas
Endpoint
GET /teorico/aulaexterna/salateorica?codigoCfcDetran={codigoCfcDetran}

Este método consulta as salas teóricas de um CFC pelo código.

4. Consulta Dados de Veículo
Endpoint
GET sincronizacaoExterna/centroFormacao/{codigoCfc}/veiculos

Este método lista todos os veículos de um CFC.

VII. Status Codes HTTP Comuns

Os seguintes códigos de status HTTP são utilizados consistentemente nas diversas APIs Credenciadas (VSoft) e de Sincronização/Consulta (Valid):

CódigoDescriçãoObjetivo/Uso
200OKEndpoint executado com sucesso e possui dados de retorno.
204NoContentEndpoint executado com sucesso, mas não possui dados de retorno.
400BadRequestParâmetros de entrada inválidos.
401UnauthorizedDados de autenticação (token) inválidos.
403ForbiddenDados de autenticação válidos, mas sem permissão.
404NotFoundA URL/Endpoint não é encontrado.
422UnprocessableEntityParâmetros de entrada válidos, mas ocorreu um erro esperado durante o processamento.
500InternalServerErrorOcorre um erro inesperado e não mapeado.
