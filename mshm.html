1. Fluxo BackOffice Exames Clínicos e Teórico

Visão Geral
O fluxo inicia com a validação biométrica na clínica e, em seguida, passa por auditoria manual e do Detran.
Etapas do Fluxo
  1. **Análise de Vivacidade**: O sistema verifica a vivacidade das biometrias para identificar fraudes.
  2. **Verificação de Fraude**:
    • **Caso sim**: O processo vai para a auditoria IH.
    • **Caso não**: É feito o envio para o SGI (Sistema de Gestão Integrada) no endpoint ValidBio com o código `1` (Biometria Validada) para o Detran.
  3. **Classificação e Auditoria**: O analista de IH e, posteriormente, o Detran, fazem a auditoria do processo.
  4. **Verificação de Aprovação do Detran**:
    • **Aprovado**: Envio para o SGI com o código `1` (Biometria Validada).
    • **Reprovado**: Envio para o SGI com o código `3` (Biometria Reprovada pelo backoffice).

2. Processamento e Envio de Resultados ao Detran

Visão Geral
Este fluxo é composto por jobs agendados que processam e enviam os resultados dos exames (Médico, Psicólogo e Teórico) em intervalos regulares.
2.1 Processamento Médico
  • **Job:** `Detran.DetranMS.ProcessingFrequenciaExameMedicoAlertaFraude` - Executa a cada **30 minutos**.
  • **Job:** `Detran.DetranMS.ProcessingFrequenciaExameMedicoAlertaValidacao` - Executa a cada **30 minutos**.
  • **Job:** `Detran.DetranMS.ProcessingFrequenciaExameMedicoAnalise` - Executa a cada **20 minutos** para alterar status do alerta.
2.2 Processamento Psicólogo
  • **Job:** `Detran.DetranMS.ProcessingFrequenciaExamePsicologicoAlertaFraude` - Executa a cada **30 minutos**.
  • **Job:** `Detran.DetranMS.ProcessingFrequenciaExamePsicologicoAlertaValidacao` - Executa a cada **30 minutos**.
  • **Job:** `Detran.DetranMS.ProcessingFrequenciaExamePsicologicoAnalise` - Executa a cada **20 minutos** para alterar status do alerta.
2.3 Processamento Exame Teórico
  • **Job:** `Detran.DetranMS.ProcessingFrequenciaExameTeoricoAlertaFraude` - Executa a cada **30 minutos**.
  • **Job:** `Detran.DetranMS.ProcessingFrequenciaExameTeoricoAlertaValidacao` - Executa a cada **30 minutos**.
  • **Job:** `Detran.DetranMS.ProcessingFrequenciaExameTeoricoAnalise` - Executa a cada **20 minutos** para alterar status do alerta.
2.4 Envio de Frequência
Jobs de Envio
Os jobs a seguir executam a cada **30 minutos** para enviar a frequência dos respectivos exames ao Detran.
  • `Detran.DetranMS.EnviarFrequenciaExameMedico`
  • `Detran.DetranMS.EnviarFrequenciaExamePsicologico`
  • `Detran.DetranMS.EnviarFrequenciaExameTeorico`

3. Sincronização de Agendamentos

3.1 Sincronização Agendamentos Psicológico

A sincronização é feita a cada **30 minutos** por um job que consulta o endpoint do Detran para obter os candidatos agendados e os dados biográficos. Em seguida, o CPF é enviado a uma fila do Rabbit para recuperar as informações biométricas.

3.2 Sincronização Agendamentos Médicos

Similar ao fluxo de exames psicológicos, um job executa a cada **30 minutos** consultando endpoints do Detran para obter agendamentos e dados biográficos. As informações biométricas são recuperadas do ABIS via fila do Rabbit.

3.3 Sincronização Agendamento Exame Teórico

A sincronização é realizada através de arquivos **.txt** em um servidor **FTP** do Detran. O job `SyncExameTeorico.SincronizarAgendamentosExameTeorico` executa a cada 30 minutos para sincronizar agendamentos de até **5 dias** para frente.

Dados FTP
**Usuário:** `valid_ftp`
**Host:** `ftp.detran.ms.gov.br`
**Porta:** `21`
**Diretório:** `\inter1`

4.2 Geração de Token DETRAN-MS

O token é gerado com base na senha (`zEpUrUPriChofL74xvalid`) e na data atual no formato `AAAAMMDD`, e é calculado usando o algoritmo **SHA1**.
